# Feature Inventory ‚Äî gsd-dashboard

Generated by `/pathfinder` Phase 0. Tauri 2 desktop app: Rust backend (`src-tauri/src/`), React+TS frontend (`src/`).

Discovery surfaced 18 fine-grained modules; the orchestrator consolidated them into **9 feature flows** for flowcharting. Domain pairs (parse + IPC query for the same data) are merged. Three concerns ‚Äî **Error Handling** (`error.rs`), **Event Broadcasting** (`events.rs`), **UI Chrome** (`App.tsx`, `components/ui/`) ‚Äî are cross-cutting and are analyzed in Phase 2 (duplication) rather than given their own flowchart.

## Feature Flows

### F1 ‚Äî Project Scan & Ingest Pipeline (backend)
- **Entry:** `src-tauri/src/commands/scan.rs:13` (`scan_projects`)
- **Core:** `scanner.rs`, `scan_roots.rs`, `scan_persistence.rs`, `scan_refresh.rs`, `scan_service.rs`, `parser/{config,plan,roadmap,state,mod}.rs`, `store/project_repo.rs`
- **Purpose:** Walk roots ‚Üí respect `.gitignore` ‚Üí discover `.planning/` ‚Üí byte-stream parse YAML into `ProjectSnapshot` ‚Üí persist to SQLite ‚Üí emit scan/project events. Per-file parse failures non-fatal.

### F2 ‚Äî Session Telemetry Pipeline (backend)
- **Entry:** `src-tauri/src/commands/sessions.rs:19` (`index_sessions`)
- **Core:** `sessions/{claude,codex,file_indexer,indexer,matcher,parallel,global,project_detail,project_charts}.rs`
- **Purpose:** JSONL streaming parse of Claude/Codex sessions with byte-offset incremental reads + partial-line tolerance ‚Üí match sessions to projects ‚Üí aggregate global/per-project ‚Üí persist.

### F3 ‚Äî File Watching & Cache Invalidation (backend)
- **Entry:** `src-tauri/src/watcher/mod.rs` (`start_watcher_service`, `refresh_session_file_for_app`)
- **Core:** `watcher/{service,roots,runtime,refresh}.rs`, `scan_roots.rs`, `scan_refresh.rs`
- **Purpose:** `notify` + debouncer watch `.planning/` subtrees + `.jsonl` files; ENOSPC ‚Üí polling fallback; emit refresh events carrying project IDs / session paths.

### F4 ‚Äî System Tray Rendering (backend)
- **Entry:** `src-tauri/src/tray/mod.rs`, wired in `bootstrap.rs` (`manage_app_state_and_tray`)
- **Core:** `tray/{service,menu,render,model}.rs`
- **Purpose:** tiny-skia PNG progress-bar icon + context menu w/ project drilldown & next-command copy; 250ms debounce; macOS template image; click routing.

### F5 ‚Äî Bootstrap & App State (backend)
- **Entry:** `src-tauri/src/main.rs:3` (`main`), `bootstrap.rs:38` (`bootstrap_app`)
- **Core:** `bootstrap.rs`, `app_state.rs`, `autostart.rs`, `updater.rs`, `store/{mod,migrations}.rs`
- **Purpose:** Plugin registration ‚Üí app data dir ‚Üí SQLite pool + WAL + migrations ‚Üí watcher launch ‚Üí tray setup ‚Üí inject Tauri-managed state.

### F6 ‚Äî Settings (end-to-end)
- **Entry:** `src-tauri/src/commands/settings.rs:17,23,29,35`; frontend `src/routes/SettingsPage.tsx`
- **Core:** `settings.rs`, `commands/settings.rs`, `store/settings_repo.rs`, `components/ScanRootsEditor.tsx`
- **Purpose:** Scan roots, hidden projects, tray sort/limit, autostart, session range; atomic writes + autostart backend sync; boot/watcher status.

### F7 ‚Äî IPC Query & Event Plumbing (frontend ‚Üî backend)
- **Entry:** `src/lib/ipc.ts`, `src/lib/appListeners.ts`, `src/App.tsx:69` (`registerAppListeners`)
- **Core:** `lib/{ipc,appListeners,queryClient,types}.ts`, backend `commands/{projects,sessions}.rs`, `events.rs`
- **Purpose:** TanStack Query wraps every `invoke()`; typed event listeners refetch on invalidation; DB-as-truth / events-as-invalidation contract.

### F8 ‚Äî Portfolio & Project Detail Views (frontend)
- **Entry:** `src/routes/PortfolioPage.tsx` (`/`), `src/routes/ProjectDetailPage.tsx` (`/project/:id`)
- **Core:** `components/{PortfolioHeaderStats,ProjectCard,ScanProgressPanel,SessionIndexProgressPanel}.tsx`, `components/charts/ActivityHeatmap.tsx`, `components/ProjectDetail/*`
- **Purpose:** Portfolio overview grid + heatmap + live progress; per-project tabbed drilldown (overview, milestones, state, charts, sessions).

### F9 ‚Äî Global Sessions View & Analytics (frontend)
- **Entry:** `src/routes/GlobalSessionsPage.tsx` (`/sessions`)
- **Core:** `components/sessions/{FilterBar,FilterChipsRow,SessionsTable}.tsx`, `lib/sessionFilters.ts`
- **Purpose:** All-sessions table w/ source/project/date filtering + sorting + copy actions + global activity charts.
